# Composer 推送产物边界清单

> 目的：回答一个具体问题: **哪些产物应该进 Git 并推送，哪些不应该。**
>
> 适用对象：`composer`、AI 协作者、人类开发者。
>
> 原则：**源码、配置模板、正式文档可以推；密钥、临时脚本、机器绑定产物、打包制品不推。**

---

## 1. 可以直接推送的产物

以下内容默认属于正式仓库资产，可正常提交与推送：

| 类别 | 示例路径 | 说明 |
|---|---|---|
| 业务源码 | `server/`、`templates/`、`app/`、`static/js/` | 直播逻辑、页面、接口、前端脚本都属于正式交付物。 |
| 环境模板 | `.env.example` | 只能包含占位符、说明、默认示例，不能包含真实密钥。 |
| 正式文档 | `docs/*.md`、模块 `README.md` | 需求、架构、部署、流程、手册、评审记录、工作报告。 |
| 边界说明 | `services/*/README.md`、`infra/*/README.md` | 即使对应服务尚未完全落地，也可作为正式设计边界文档入库。 |
| 正式部署定义 | `docker-compose.prod.yml`、`Dockerfile` | 前提是内容面向团队通用部署，而不是某台机器的临时快照。 |

---

## 2. 默认禁止推送的产物

以下内容默认 **不得** 提交，除非先完成脱敏、重构或得到明确批准：

| 类别 | 当前仓库中的例子 | 原因 |
|---|---|---|
| 明文密码脚本 | `connect_server.exp`、`ssh_with_pass.exp`、`ssh_password_only.exp`、`test_ssh.py` | 含真实口令或等同口令。 |
| 打包制品 | `usking-deploy.tar.gz` | 可重建、体积大、会过期，不属于源码。 |
| 时间戳部署快照 | `deploy-20260325-172328/` | 属于一次性本地产物，容易与正式部署入口分叉。 |
| 本机路径或机器绑定信息 | 文档中的 `/Users/...` 绝对路径、单机目录、个人用户名 | 远端不可用，且暴露本地工作区习惯。 |
| 真实环境文件 | `.env`、数据库、上传目录、证书 | 属于运行时私有数据。 |

---

## 3. 需要整理后再考虑推送的产物

以下内容不是绝对禁止，但必须先转化为**团队可复用、无敏感信息、唯一可信入口**：

| 产物 | 当前状态 | 入库前要求 |
|---|---|---|
| `deploy.sh` | 本地部署辅助脚本 | 去除机器假设，避免复制 `.env` 实值，明确与 `docs/DEPLOY.md` 的关系。 |
| 服务器初始化脚本 | 目前散落在部署快照目录 | 收敛到统一目录，如未来的 `deploy/`。 |
| Nginx 配置样例 | 与快照目录绑定 | 提炼为通用模板，使用占位符而非具体服务器快照。 |
| 生产 compose 变体 | 存在快照目录副本 | 只能保留一个 canonical 版本，避免多份长期分叉。 |

---

## 4. Composer 执行规则

`composer` 在准备提交前，必须按以下顺序检查：

1. 先区分本次改动属于 **源码 / 正式文档 / 临时产物 / 敏感信息** 哪一类。
2. 若文件含有以下任一特征，默认不推：
   - 明文密码
   - `.env` 实值
   - 服务器 IP + 登录凭据组合
   - 本地时间戳快照目录
   - 打包产物
3. 若是部署脚本，必须确认：
   - 是否能被团队其他人直接复用
   - 是否依赖本机路径
   - 是否重复已有 `docs/DEPLOY.md` 或 `docker-compose.prod.yml`
4. 若是文档，必须确认：
   - 使用仓库相对路径
   - 不引用本机绝对路径
   - 不把“规划中/占位”写成“已上线”
5. 提交说明中应明确写出：
   - 本次包含哪些正式产物
   - 明确排除了哪些本地或敏感文件

## 5. 当前仓库的建议边界

### 5.1 现在可以放心继续推的

- `server/`
- `templates/`
- `app/`
- `static/js/`
- `docs/`
- `.env.example`
- `README.md`
- `services/*/README.md`
- `infra/*/README.md`

### 5.2 现在不要推的

- `*.exp`
- `*.tar.gz`
- `deploy-*/`
- `test_ssh.py`
- `.env`
- `static/uploads/*`
- `*.db`
- `*.pem`

### 5.3 以后若要推，先重构

- `deploy.sh`
- 任意“只适配一台服务器”的脚本
- 含固定公网 IP、固定域名、固定路径的部署快照
